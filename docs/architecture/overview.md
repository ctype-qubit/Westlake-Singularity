# 🏗️ Westlake Singularity 架构总览

## 一、设计哲学

**"实验室即科学家"** —— 不是把 AI 塞进现有实验室，而是围绕 AI 重建实验室。

核心原则：
1. **Agent First** — 每个仪器、传感器、计算节点都是一个 Agent
2. **Asynchronous Swarm** — 千个 Agent 通过消息异步协作，无中心瓶颈
3. **Verification Loop** — 每个AI结论都必须经过独立验证路径
4. **Physics-Grounded** — 所有操作都受物理约束（量纲检查、能量守恒、不确定性量化）

---

## 二、六层架构详解

### 层1 · 物理层
量子比特硬件：FeTeSe 铁基超导体 + hBN 封装 + 量子点 + 量子电容读出
- 工作温度: ~20 mK
- 6线圈磁场控制（4角+2交换）
- Majorana编织序列

### 层2 · 感知层
多模态传感器阵列：
- 8K 工业相机 (全局快门, 1μm/pixel)
- 激光雷达 (LiDAR, 实时3D点云)
- 麦克风阵列 (超声到次声)
- 温湿度/振动/电磁场传感器

每个传感器都是一个 `Guard Agent` 的输入通道。

### 层3 · 控制层
- **STM 控制**: Nanonis API (gRPC → Python binding)
- **超算调度**: Slurm REST API → 自动作业提交/监控
- **硬件控制**: Arduino/LabVIEW 抽象层
- **实时流**: WebSocket 双向通讯，毫秒级反馈

### 层4 · 认知层
三种认知 Agent 协同：
- **Guard (守护者)**: 异常检测、安全阈值、心跳监控
- **Mapper (映射者)**: STM参数优化、图像质量闭环
- **Discovery (发现者)**: 相变检测、拓扑信号识别

所有认知 Agent 通过 **Orchestrator (中控)** 协调，形成：
```
观测 → 假设 → 验证 → 批判 → 迭代
```

### 层5 · 数字孪生
虚拟-现实对齐：
- COMSOL Multiphysics 仿真 → 实验参数预测
- DFT (VASP/QE) 计算 → 能带结构验证
- Sim-to-Real: 仿真结果反馈实际实验

### 层6 · 联邦帝国
多实验室 Agent 联邦（WAP 协议）：
- **组内联邦**: 同一课题组内 Agent 间自由通讯
- **跨组外交**: 不同实验室 Agent 经 WAP 网关通信
- **匿名数据共享**: 脱敏后共享实验结果
- **联邦学习**: 多实验室联合训练物理模型

## 四、安全模型

| 威胁 | 防护 |
|------|------|
| Agent 幻觉导致危险操作 | Guard 硬阈值拦截 + 物理约束验证 |
| 跨组数据泄露 | WAP 数据脱敏 + 联邦加密隧道 |
| 单点故障 | Orchestrator 热备 + 心跳选举 |
| 超算资源耗尽 | Slurm QoS + 作业优先级队列 |
